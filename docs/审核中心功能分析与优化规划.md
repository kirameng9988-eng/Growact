# 审核中心功能分析与优化规划

> 更新时间：2024-01-15

---

## 一、审核中心的用途

### 1.1 核心定位
- **内容质量控制**：确保待发布内容符合规范
- **审核流程管理**：管理内容的审核状态流转
- **审核历史追溯**：记录审核操作及原因

### 1.2 审核状态流转
```
草稿 → 待审核 → 审核中 → 已发布
                ↓
              已驳回 → 待审核（重新提交）
```

### 1.3 当前审核结构
```
待审核 (12) | 审核中 (28) | 已通过 (486) | 已驳回 (15)
```

## 二、当前功能差距分析

### 2.1 功能清单

| 功能 | 状态 | 说明 |
|-----|------|------|
| 审核统计卡片 | ⚠️ 静态 | 数字硬编码，无动态更新 |
| Tab切换 | ❌ 无效 | 按钮无交互，状态不变 |
| 分类筛选 | ❌ 无效 | 下拉框无交互 |
| 审核列表 | ⚠️ 静态 | HTML硬编码4条 |
| 通过操作 | ⚠️ 假 | 只显示Toast |
| 驳回操作 | ⚠️ 假 | 弹窗无关联内容 |
| 预览功能 | ❌ 无效 | 按钮无交互 |
| 分页 | ❌ 无效 | 按钮无交互 |
| 批量通过 | ❌ 无效 | 按钮无交互 |
| 批量拒绝 | ❌ 无效 | 按钮无交互 |
| 侧边栏Badge | ⚠️ 静态 | 固定12，无联动 |

### 2.2 核心问题

1. **数据静态化**：所有审核项都是HTML硬编码
2. **状态无关联**：驳回弹窗无法指定具体内容
3. **操作无效**：通过/驳回只显示Toast，无实际状态变更
4. **无预览**：预览按钮无响应

## 三、优化规划

### 3.1 功能重构

**审核统计**
- 动态计算：待审核、已审数、审核中、已驳回
- Tab按钮显示对应数量
- 点击Tab过滤列表

**审核列表**
- 动态渲染审核项
- 按状态筛选显示
- 支持分类筛选
- 分页加载

**审核操作**
- 通过：改变状态为已通过，移出待审核列表
- 驳回：弹出原因选择，输入说明，更新状态为已驳回
- 预览：弹出详情弹窗查看完整内容

**批量操作**
- 批量通过：选中多项一次性通过
- 批量拒绝：弹出原因输入，一次性驳回多项
