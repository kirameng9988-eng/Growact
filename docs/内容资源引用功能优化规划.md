# 内容资源引用功能优化规划

> 更新时间：2024-01-15

---

## 一、需求概述

### 1.1 问题现状
- 当前创建内容时，封面图和正文插入图片都是简单的本地上传
- 资源中心（图片/文件/视频）与内容创建完全独立，没有关联
- 无法复用已上传的资源，造力资源重复上传

### 1.2 优化目标
- 建立资源中心与内容创建的引用关系
- 支持从资源中心选择已有资源
- 富文本编辑器支持插入资源中心的图片/文件/视频

---

## 二、功能架构

```
资源中心（Asset）
       ↓
被内容引用（Reference）
       ↓
内容展示（Content）
       ↓
门户消费（Portal）
```

## 三、封面图选择优化

### 3.1 交互设计

**入口区域（原"上传图片"位置）：**

```
┌─────────────────────────────────────────────────────┐
│  📌 选择资源 / 上传资源                              │
│                                                     │
│  ┌─────────────────┐  ┌─────────────────┐         │
│  │  🖼️ 从资源中心选择 │  │  ⬆️ 上传新资源    │         │
│  │                  │  │                  │         │
│  │  已有封面图库     │  │  直接本地上传     │         │
│  └─────────────────┘  └─────────────────┘         │
└─────────────────────────────────────────────────────┘
```

**从资源中心选择：**
- 点击打开资源选择弹窗
- 支持图片库列表展示
- 支持搜索、筛选（分类、时间）
- 支持预览大图
- 选中后返回选中图片

**上传新资源：**
- 保持原有上传功能
- 上传成功后自动添加到资源中心

### 3.2 组件设计

**资源选择弹窗（ResourcePickerModal）：**

| 功能 | 说明 |
|-----|------|
| 标签页 | 图片 / 文件 / 视频 |
| 搜索 | 按名称搜索资源 |
| 筛选 | 按上传时间、分类筛选 |
| 列表 | 网格/列表视图切换 |
| 预览 | 鼠标悬停显示大图预览 |
| 分页 | 加载更多/分页 |

**选择模式：**
- 单选模式（封面图）
- 多选模式（正文插入）

## 四、正文编辑器资源插入优化

### 4.1 工具栏新增按钮

在现有工具栏基础上增加：

```
[B] [I] [U] [|] [列表] [编号] [链接] [图片] [📁资源]
```

**新增 📁资源 按钮：**
- 图标：`fa-folder-open` 或自定义资源图标
- 颜色：与图片按钮一致
- 点击弹出资源选择器


### 4.3 插入展示方式

| 资源类型 | 插入效果 | HTML渲染 |
|---------|---------|---------|
| 图片 | inline行内展示 | `<img src="url" class="resource-image">` |
| 文件 | 卡片链接样式 | `<div class="resource-file-card">📄 文件名.pdf (2.3MB)</div>` |
| 视频 | 视频播放组件 | `<video src="url" controls>` 或嵌入播放组件 |

**图片插入示例：**
```html
<img src="https://cdn.example.com/images/meeting.jpg"
     class="resource-image"
     data-resource-id="12345"
     alt="会议现场">
```

**文件插入示例：**
```html
<div class="resource-file-card" data-resource-id="12346">
    <i class="fas fa-file-pdf"></i>
    <div class="resource-file-info">
        <span class="resource-file-name">年度工作报告.pdf</span>
        <span class="resource-file-size">2.3MB</span>
    </div>
    <a href="https://cdn.example.com/files/report.pdf" class="resource-file-download">
        <i class="fas fa-download"></i> 下载
    </a>
</div>
```

**视频插入示例：**
```html
<video class="resource-video"
       controls
       data-resource-id="12347">
    <source src="https://cdn.example.com/videos/intro.mp4" type="video/mp4">
</video>
```

## 五、API 设计

### 5.1 资源查询接口

```
GET /api/resource/list
```

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| type | string | 否 | 资源类型：image/file/video |
| keyword | string | 否 | 搜索关键字 |
| page | int | 否 | 页码 |
| pageSize | int | 否 | 每页条数 |

**响应示例：**
```json
{
    "code": 200,
    "data": {
        "total": 156,
        "list": [
            {
                "id": "1",
                "name": "meeting.jpg",
                "type": "image",
                "url": "https://cdn.example.com/images/meeting.jpg",
                "size": "2.3MB",
                "width": 1920,
                "height": 1080,
                "createdAt": "2024-01-15 10:30:22"
            }
        ]
    }
}
```

### 5.2 资源选择弹窗组件接口

**前端组件属性：**
```javascript
// ResourcePicker Modal Props
{
    visible: boolean,           // 是否显示
    mode: 'single' | 'multiple',// 选择模式
    resourceType: 'image' | 'file' | 'video' | 'all',
    onSelect: (resources) => {}, // 选择回调
    onClose: () => {},          // 关闭回调
}
```

## 六、数据库设计

### 6.1 内容资源关联表（可选）

如果需要追踪资源被引用情况：

```sql
CREATE TABLE `content_resource_ref` (
    `id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `content_id` BIGINT(20) NOT NULL COMMENT '内容ID',
    `resource_id` BIGINT(20) NOT NULL COMMENT '资源ID',
    `ref_type` VARCHAR(20) NOT NULL COMMENT '引用类型：cover/content',
    `ref_location` VARCHAR(50) NULL COMMENT '引用位置',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_content_id` (`content_id`),
    INDEX `idx_resource_id` (`resource_id`)
) ENGINE=INNODB DEFAULT CHARSET=utf8mb4 COMMENT='内容资源引用表';
```

### 6.2 说明
- 当前版本可以不做强关联，通过资源URL直接引用
- 后续可根据追踪需求决定是否增加关联表
