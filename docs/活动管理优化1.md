【创建活动、查看活动、编辑活动页面布局修复】

发现页面存在布局问题，需要统一修复。

=========================================
问题1：面包屑重复
=========================================

当前页面出现：

活动运营 / 活动管理

活动运营 / 活动管理 / 创建活动

说明：

Layout已经渲染面包屑。

activity-create.html
activity-edit.html
activity-view.html

内部又写了一套面包屑。

要求：

删除页面内部面包屑。

统一使用Layout生成面包屑。

规则：

活动管理
面包屑：

活动运营 / 活动管理

创建活动
面包屑：

活动运营 / 活动管理 / 创建活动

编辑活动
面包屑：

活动运营 / 活动管理 / 编辑活动

活动详情
面包屑：

活动运营 / 活动管理 / 活动详情

由Layout动态生成。

页面内部不得再出现Breadcrumb组件。

=========================================
问题2：内容区域左侧留白过大
=========================================

当前创建活动页面：

左侧出现大面积空白。

原因：

页面内部仍保留旧布局代码：

margin-left
padding-left
content-offset
sidebar-offset

等样式。

Layout已经负责菜单布局。

页面不允许再次预留菜单宽度。

检查：

activity-create.html

activity-edit.html

activity-view.html

删除：

margin-left: 220px
padding-left: 220px
sidebar-width
content-offset

类似代码。

=========================================
问题3：页面宽度自适应
=========================================

创建活动页面改为：

width: 100%

占满主内容区域。

不要固定：

max-width:1200px

不要居中：

margin:0 auto

改为：

width:100%

让表单跟随内容区域自适应。

=========================================
问题4：返回列表按钮
=========================================

保留右上角：

返回列表

按钮。

位置：

页面标题右侧。

点击：

返回活动管理列表。

=========================================
问题5：统一规范
=========================================

以下页面全部检查：

activity-list.html

activity-create.html

activity-edit.html

activity-view.html

activity-audit.html

activity-publish.html

确保：

页面只负责业务内容。

顶部Header
左侧菜单
面包屑

全部由Layout统一管理。

禁止页面再次嵌套Layout。
