# 内容创建页面 - 分类/标签下拉联动规划

> 更新时间：2024-01-15

---

## 一、需求分析

### 1.1 当前问题
- 内容创建页面的分类下拉、标签输入均为静态数据
- 分类管理和标签管理没有与内容创建形成联动
- 无法实时同步分类/标签管理页面的变更

### 1.2 目标
- 内容创建/编辑页面的**分类下拉**与**分类管理**联动
- 内容创建/编辑页面的**标签输入**与**标签管理**联动
- 当分类/标签在管理页面新增/编辑/删除后，内容创建页面自动更新

---

## 二、联动方案设计

### 2.1 数据共享机制

由于是纯前端原型，采用**LocalStorage** 实现数据共享：

```javascript
// 存储结构
localStorage.setItem('content_categories', JSON.stringify([...]))
localStorage.setItem('content_tags', JSON.stringify([...]))
```

| 存储Key | 数据内容 | 更新时机 |
|--------|---------|---------|
| `content_categories` | 分类树形数据 | 分类管理增删改 |
| `content_tags` | 标签列表数据 | 标签管理增删改 |

### 2.2 分类联动

**分类管理页面**
- 新增/编辑/删除分类时，同步更新 `localStorage.content_categories`
- 提供 `syncCategories()` 函数供其他页面调用

**内容创建/编辑页面**
- 页面加载时从 `localStorage` 读取分类数据
- 初始化分类下拉选项
- 数据结构：
```javascript
const categories = [
  { id: 1, name: '新闻资讯', children: [
    { id: 11, name: '时政要闻' },
    { id: 12, name: '社会民生' }
  ]},
  { id: 2, name: '政策文件', children: [...] }
]
```

### 2.3 标签联动

**标签管理页面**
- 新增/编辑/合并/删除标签时，同步更新 `localStorage.content_tags`
- 提供 `syncTags()` 函数供其他页面调用

**内容创建/编辑页面**
- 页面加载时从 `localStorage` 读取标签数据
- 标签输入改为**下拉多选**或**可搜索标签选择器**
- 提供**创建新标签**的快捷入口
- 数据结构：
```javascript
const tags = [
  { id: 1, name: '时政', color: 'accent-blue', count: 128 },
  { id: 2, name: '经济', color: 'accent-green', count: 86 }
]
```

## 四、标签输入组件设计

### 4.1 组件形态
- **可搜索多选下拉**
- 输入关键词过滤已有标签
- 按回车创建新标签
- 已选标签以标签_chip形式展示

### 4.2 交互流程
```
用户输入"两会" →
  1. 显示匹配的已有标签列表
  2. 如果没有匹配，显示"创建标签: 两会"选项
  3. 用户点击或按回车 →
     - 如果标签存在：选中
     - 如果标签不存在：调用标签管理同步接口创建
```

### 4.3 快捷创建
- 输入不存在的标签名后，按回车
- 自动调用 `syncTags()` 创建新标签
- 新标签自动选中和显示

## 五、验收标准

### 5.1 分类联动
- [ ] 分类管理新增分类后，内容创建页面下拉可选
- [ ] 分类管理删除分类后，内容创建页面下拉不再显示
- [ ] 分类管理修改分类名称后，内容创建页面同步更新

### 5.2 标签联动
- [ ] 标签管理新增标签后，内容创建页面可搜索到
- [ ] 标签管理删除标签后，内容创建页面不可选
- [ ] 内容创建页面可直接创建新标签并选中
- [ ] 标签管理合并标签后，内容创建页面显示合并后的标签

### 5.3 数据一致性
- [ ] 页面刷新后数据保持一致
- [ ] 多个页面同时打开时数据同步
