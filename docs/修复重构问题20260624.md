# 修复内容服务平台双框架嵌套问题（高优先级）

## 当前问题

当前系统已经采用：

index.html

作为统一入口。

包含：

- 顶部导航 Header
- 左侧菜单 Sidebar
- 面包屑 Breadcrumb
- 内容区域 Content

但是部分业务页面在加载后：

页面内部又包含了一套：

- Header
- Sidebar
- 菜单导航
- 页面框架

导致出现：

【框架套框架】

具体表现：

- 顶部导航出现两次
- 左侧菜单出现两次
- 页面结构混乱
- 演示效果较差

截图中活动分类页面已经出现该问题。

---

## 目标架构

采用：

Single Layout Architecture

整个系统只允许存在一个公共框架。

结构如下：

MainLayout

├─ Header
├─ Sidebar
├─ Breadcrumb
└─ Content

---

## Header规范

顶部导航仅允许存在于：

index.html

禁止任何业务页面再次出现：

<header>
topbar
navbar

相关代码。

发现后全部移除。

---

## Sidebar规范

左侧菜单仅允许存在于：

index.html

禁止任何业务页面再次出现：

<aside>
sidebar
menu

相关代码。

发现后全部移除。

---

## Breadcrumb规范

面包屑统一由：

index.html

维护。

业务页面禁止再次渲染面包屑。

---

## 页面改造原则

所有业务页面：

只保留业务内容。

例如：

activity-category.html

最终结构：

<div class="page-container">

  <div class="page-header">
      页面标题
  </div>

  <div class="page-toolbar">
      查询区域
      操作按钮
  </div>

  <div class="page-content">
      表格
      图表
      卡片
  </div>

</div>

禁止包含：

<html>
<head>
<body>
<header>
<aside>
sidebar
navbar
layout

等公共框架代码。

---

Header

Layout
Breadcrumb

发现后全部删除。

---

## 路由规范

所有菜单点击：

统一通过：

loadPage(pageKey)

进行切换。

只更新：

Content区域。

禁止：

window.location跳转

禁止：

打开新页面

禁止：

刷新整个页面

---

## 页面切换效果

点击菜单后：

Header保持不变

Sidebar保持不变

Breadcrumb更新

Content区域更新

整体效果参考：

- 腾讯云控制台
- 阿里云控制台
- Ant Design Pro
- 若依 Pro

---

## 验收标准

必须满足：

✓ 页面顶部导航只出现一次

✓ 左侧菜单只出现一次

✓ 面包屑只出现一次

✓ 页面切换不刷新

✓ 所有菜单正常打开

✓ 所有原有功能页面保留

✓ 所有 Mock 数据保留

✓ 不重写业务页面

✓ 不修改现有UI风格

✓ 仅修复框架嵌套问题

最终效果：

整个系统只有一套公共框架，所有业务页面都在内容区域中切换展示，不再出现菜单套菜单、导航套导航的问题。
