# 内容服务平台菜单重组与导航优化（Demo封版版）

## 开发目标

当前内容服务平台已经完成：

* 工作台
* 内容中心相关页面
* 活动运营相关页面
* 资源中心
* 数据分析
* 系统管理
* 日志管理

大部分页面已经开发完成。

本次任务：

不是重做页面。

不是重做UI。

不是重做功能。

而是：

> 基于现有页面重新组织菜单结构与导航架构。

目标：

让系统看起来像：

> 一个完整的运营平台

而不是多个页面拼接。

---

# 重要要求

禁止：

* 删除已有页面
* 重写已有页面
* 重构页面业务逻辑
* 修改页面UI风格

允许：

* 调整菜单结构
* 调整菜单顺序
* 调整页面归属
* 修改路由映射
* 修改导航逻辑
* 重组 portal.html

最终：

所有页面仍然可访问。

所有按钮仍然可点击。

所有 Mock 数据保留。

处理方式：

保留页面文件。

```text

```

# 导航重构要求

当前问题：

部分页面拥有自己的菜单。

导致：

* 菜单重复
* 菜单丢失
* 导航混乱

统一改造：

采用：

```text
Header
+
Sidebar
+
Content
```

单壳层模式。

---

整个系统：

只允许：

一个 Sidebar。

禁止：

页面内部再次出现左侧菜单。

---

# portal.html重构

要求：

portal.html 成为唯一入口。

所有菜单点击：

通过：

```javascript
loadPage(pageKey)
```

切换内容。

不要刷新页面。

不要跳转新窗口。

---

支持：

```text
#/dashboard

#/activity/manage

#/activity/category

#/activity/sign

#/resource/image

#/system/user
```

Hash路由。

刷新可恢复。
