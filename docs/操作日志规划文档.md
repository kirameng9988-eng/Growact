# 内容服务平台操作日志清单

> 本文档用于指导开发团队梳理和规划系统操作日志功能，满足等保2.0审计要求。
>
> 更新时间：2024-01-15

## 一、日志模块概述

### 1.1 设计目的
- 满足等保2.0安全审计要求
- 实现系统操作可追溯
- 便于问题排查和安全分析

### 1.2 日志分类
| 日志类型 | 说明 | 记录内容 |
|---------|------|---------|
| 登录日志 | 用户登录/退出系统 | 登录行为、时间、结果、设备信息 |
| 操作日志 | 业务操作行为 | 新增/编辑/删除/发布/审核等操作 |

### 1.3 记录字段（统一结构）
| 字段名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| 操作用户 | string | 执行操作的用户名 | 张明远 |
| 用户角色 | string | 用户所属角色 | 超级管理员 |
| 操作模块 | string | 被操作的功能模块 | 内容管理 |
| 操作行为 | enum | 操作类型 | 新增/编辑/删除/发布/登录等 |
| 操作内容描述 | string | 具体操作说明 | 编辑文章《2024年度工作计划》 |
| IP地址 | string | 操作者IP | 192.168.1.100 |
| 设备信息 | string | 浏览器/操作系统 | Chrome/Win10 |
| 操作时间 | datetime | 操作时间戳 | 2024-01-15 10:30:22 |
| 操作结果 | enum | 成功/失败 | 成功/失败 |

## 四、接口设计

### 4.1 日志记录接口

```
POST /api/log/record
```

**请求参数：**
```json
{
    "userId": "string",           // 用户ID
    "userName": "string",         // 用户名
    "userRole": "string",         // 用户角色
    "module": "string",           // 操作模块
    "action": "string",           // 操作行为
    "description": "string",      // 操作描述
    "ipAddress": "string",        // IP地址
    "deviceInfo": "string",       // 设备信息
    "operationTime": "datetime", // 操作时间
    "result": "string",           // 操作结果
    "logType": "string"           // 日志类型：login/operation
}
```

### 4.2 日志查询接口

```
GET /api/log/list
```

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| logType | string | 否 | 日志类型：login/operation |
| keyword | string | 否 | 关键字搜索 |
| startDate | string | 否 | 开始日期 |
| endDate | string | 否 | 结束日期 |
| page | int | 否 | 页码 |
| pageSize | int | 否 | 每页条数 |

**响应示例：**
```json
{
    "code": 200,
    "data": {
        "total": 156,
        "list": [
            {
                "id": "1",
                "userName": "张明远",
                "userRole": "超级管理员",
                "module": "内容管理",
                "action": "编辑",
                "description": "编辑文章《2024年度工作计划》",
                "ipAddress": "192.168.1.100",
                "deviceInfo": "Chrome/Win10",
                "operationTime": "2024-01-15 10:30:22",
                "result": "成功"
            }
        ]
    }
}
```

### 4.3 日志导出接口

```
GET /api/log/export
```

### 4.4 日志清空接口

```
DELETE /api/log/clear
```

## 五、数据库表设计

### 5.1 system_log 表

```sql
CREATE TABLE `system_log` (
    `id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `log_type` VARCHAR(20) NOT NULL DEFAULT 'operation' COMMENT '日志类型：login-登录日志，operation-操作日志',
    `user_id` BIGINT(20) NULL COMMENT '用户ID',
    `user_name` VARCHAR(100) NOT NULL COMMENT '用户名',
    `user_role` VARCHAR(50) NULL COMMENT '用户角色',
    `module` VARCHAR(50) NOT NULL COMMENT '操作模块',
    `action` VARCHAR(50) NOT NULL COMMENT '操作行为',
    `description` TEXT NULL COMMENT '操作描述',
    `ip_address` VARCHAR(50) NULL COMMENT 'IP地址',
    `device_info` VARCHAR(200) NULL COMMENT '设备信息',
    `operation_time` DATETIME NOT NULL COMMENT '操作时间',
    `result` VARCHAR(20) NOT NULL DEFAULT '成功' COMMENT '操作结果：成功/失败',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_log_type` (`log_type`),
    INDEX `idx_user_name` (`user_name`),
    INDEX `idx_module` (`module`),
    INDEX `idx_operation_time` (`operation_time`)
) ENGINE=INNODB DEFAULT CHARSET=utf8mb4 COMMENT='系统操作日志表';
```

### 5.2 日志保留策略

| 环境 | 保留期限 | 说明 |
|-----|---------|------|
| 生产环境 | 6个月 | 等保要求至少保留6个月 |
| 测试环境 | 30天 | 便于问题排查 |

## 六、前端页面交互

### 6.1 页面路径
- 页面文件：`/system-log.html`
- 路由：`/system/log`

### 6.2 功能交互

| 功能 | 交互说明 | 特殊处理 |
|-----|---------|---------|
| Tab切换 | 点击登录日志/操作日志切换 | 仅显示对应类型数据 |
| 搜索 | 关键字搜索用户/模块/行为 | 支持模糊匹配 |
| 时间筛选 | 选择开始/结束日期 | 格式：YYYY-MM-DD |
| 分页 | 上一页/下一页/跳转 | 每页默认20条 |
| 导出 | 导出当前筛选条件日志 | 提示"导出中..." |
| 清空 | 清空所有日志 | 需二次确认弹窗 |

### 6.3 权限控制

| 角色 | 查看日志 | 导出日志 | 清空日志 |
|-----|---------|---------|---------|
| 超级管理员 | ✓ | ✓ | ✓ |
| 管理员 | ✓ | ✓ | ✗ |
| 普通用户 | ✗ | ✗ | ✗ |
