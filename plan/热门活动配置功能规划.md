# 热门活动配置功能规划

## 一、需求分析

### 现状
- 门户页面（activity-zone/index.html）中的热门活动区域数据是硬编码的Mock数据
- 管理员无法通过后台界面配置哪些活动要展示在门户热门区域

### 目标
- 在活动运营中心增加「热门活动配置」功能
- 管理员可以设置哪些活动展示在门户的热门活动区域
- 支持设置展示顺序、展示位置（首页/列表页等）

## 二、方案设计

### 方案A：在活动管理中增加"推荐到门户"功能

在现有活动管理页面（activity-list.html）的活动列表中，增加一列"门户推荐"，管理员可以：
1. 给活动打上"热门推荐"标记
2. 设置推荐权重（用于排序）
3. 设置展示位置（首页精选/列表页置顶等）

**优点：** 简单直接，不需要新增页面
**缺点：** 所有活动混在一起，不便于批量管理热门活动

### 方案B：独立热门活动配置页面

新增一个「热门活动配置」页面（hot-activity-config.html），专门管理：
1. 首页热门活动列表（最多展示N个）
2. 列表页置顶活动
3. 推荐权重排序
4. 活动时间范围（自动上下线）

**优点：** 功能清晰，便于批量管理
**缺点：** 需要新增页面

### 推荐方案B

理由：
1. 热门活动配置是独立功能，独立页面更清晰
2. 后续可能需要配置首页Banner、精选活动等，扩展性好
3. 门户运营通常需要整体规划首页内容

## 三、功能规划

### 3.1 热门活动配置页面（hot-activity-config.html）

**功能点：**
1. **热门活动列表**
   - 表格展示已配置为热门的所有活动
   - 字段：活动名称、推荐位置、权重、开始时间、结束时间、状态、操作
   - 支持拖拽排序

2. **添加热门活动**
   - 点击"添加"按钮弹出选择器
   - 从已有活动列表中选择（支持搜索）
   - 设置推荐位置（首页精选/列表页置顶）
   - 设置展示权重（数字越大越靠前）
   - 设置展示时间范围

3. **编辑热门活动**
   - 修改推荐位置、权重、时间

4. **移除热门**
   - 将活动从热门列表移除（不是删除活动）

5. **批量操作**
   - 批量设置展示时间
   - 批量移除

**Mock数据：** 至少5条热门活动配置

### 3.2 活动管理页面改造（activity-list.html）

在活动列表增加"门户推荐"列：
- 显示当前活动的推荐状态（首页精选/列表置顶/普通）
- 快速操作：设为首页精选/取消推荐

### 3.3 门户页面改造（activity-zone/index.html）

修改热门活动数据来源：
- 原来：从hardcode的mockData读取
- 改为：从LocalStorage读取配置的活动数据

## 六、开发顺序

1. **第一步：** 创建热门活动配置页面（hot-activity-config.html）
2. **第二步：** 改造活动管理页面，增加门户推荐列
3. **第三步：** 改造门户页面，读取配置而非hardcode

## 七、优先级

- 建议先完成配置页面，确保功能完整性
- 活动列表改造可以后期迭代
- 门户页面需要同步改造才能看到效果
