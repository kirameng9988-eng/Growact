# 《内容中台角色权限规划方案》

## 一、设计原则

参考腾讯云、阿里云、Ant Design Pro 等成熟后台系统的权限模型，采用 **RBAC（基于角色的访问控制）** 模型：

```
角色 → 权限集合 → 用户
```

权限结构：**分类 → 功能菜单 → 操作按钮**

## 四、页面结构设计

```
┌─────────────────────────────────────────────────────────────┐
```

## 五、交互设计

### 5.1 权限配置交互

| 交互 | 行为 |
|------|------|
| 勾选菜单 | 自动勾选默认「查看」权限 |
| 取消菜单 | 自动取消该菜单下所有按钮权限 |
| 勾选全选 | 勾选当前菜单所有权限（菜单+所有按钮） |
| 取消全选 | 取消当前菜单所有权限 |
| 保存成功 | Toast 提示，返回角色列表 |
| 返回 | 直接返回角色列表（不保存） |

## 六、角色预设方案

| 角色 | 角色编码 | 权限范围 | 说明 |
|------|---------|---------|------|
| 超级管理员 | admin | 全部权限 | 拥有系统所有权限 |
| 内容编辑 | editor | 内容中心-部分 + 资源中心 | 可创建、编辑、提交审核 |
| 审核员 | reviewer | 内容中心-审核 + 发布 | 可审核内容、发布/下线 |
| 运营人员 | operator | 门户运营全部 | 可装修首页、管理Banner/专题 |
| 数据分析师 | analyst | 数据分析全部 + 查看 | 可查看和导出数据 |
| 访客 | guest | 工作台 + 内容查看 | 只读权限 |

## 七、UI规范

### 7.1 配色保持

保持当前政务科技蓝风格：
- 主色：`#3b82f6` (accent-blue)
- 背景：`#030714` (primary-bg)、`#0a1628` (card-bg)
- 边框：`#1a2744` (border-dark)
- 文字：`#f1f5f9` (text-primary)、`#94a3b8` (text-secondary)

### 7.2 权限配置表格样式

```
分类列：120px，字体加粗，带左侧色条标识
菜单列：200px，含复选框
按钮列：自适应，Tag Checkbox 布局
操作列：80px，全选复选框居中
```

### 7.3 Tag Checkbox 样式

```
[☑ 查看] [☑ 新增] [☐ 编辑] [☐ 删除] [☑ 发布]
```

间距 8px，圆角 6px，选中时背景高亮。
